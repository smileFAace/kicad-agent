**Test class: TestCrossFilePartialFailure**

6. `test_partial_failure_rollback_both_files`:
   - Copy sch + pcb to tmp_path
   - Record original file contents (read both files before operation)
   - This test is tricky because we need a real failure mid-operation. The approach:
     - Use monkeypatch to make `propagate_footprint_ref` raise ValueError after `propagate_symbol_ref` succeeds
     - Create Operation targeting both files
     - Call executor.execute(op) -- expect it to raise (or return error)
     - Verify BOTH files are unchanged from original (AtomicOperation rollback worked)
   - Implementation: use `monkeypatch.setattr("kicad_agent.ops.executor.propagate_footprint_ref", ...)` inside the handler module to inject a failure. The handler imports propagate_footprint_ref lazily, so monkeypatch needs to target the right namespace. Alternatively, use a schema validation that will fail on one file but not the other.
   - SIMPLEST approach: Create a test where old_lib_id contains a null byte for the PCB-specific call. Since propagation.py validates against null bytes, the PCB propagation will fail. But wait -- the handler calls propagate_symbol_ref for schematics and propagate_footprint_ref for PCBs, both with the same old_lib_id. So if old_lib_id is valid, both pass; if invalid, both fail.
   - BETTER approach: Use monkeypatch to inject failure:
     ```python
     def test_partial_failure_rollback_both_files(self, project_dir, monkeypatch):
         sch = project_dir / "Arduino_Mega.kicad_sch"
         pcb = project_dir / "Arduino_Mega.kicad_pcb"
         original_sch = sch.read_text()
         original_pcb = pcb.read_text()

         # Inject failure into propagate_footprint_ref only
         from kicad_agent.crossfile import propagation
         original_fp_ref = propagation.propagate_footprint_ref

         def failing_fp_ref(*args, **kwargs):
             raise ValueError("Injected failure for testing")

         monkeypatch.setattr(propagation, "propagate_footprint_ref", failing_fp_ref)

         op = Operation(root=PropagateSymbolChangeOp(...))
         executor = OperationExecutor(base_dir=project_dir)

         # The exception should propagate from the handler through AtomicOperation
         # which auto-rolls back both files
         with pytest.raises(ValueError, match="Injected failure"):
             executor.execute(op)

         # Verify BOTH files are restored to original state
         assert sch.read_text() == original_sch
         assert pcb.read_text() == original_pcb
     ```
