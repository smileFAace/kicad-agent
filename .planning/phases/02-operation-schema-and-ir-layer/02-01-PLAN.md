Create `src/kicad_agent/ops/schema.py` with the following structure:

1. **Module docstring** following project convention (purpose, design decisions, usage example).

2. **Shared field types** (Pydantic BaseModel subclasses):
   - `PositionSpec(BaseModel)` with fields: `x: float`, `y: float`, `angle: float = 0.0`
   - `PropertySpec(BaseModel)` with fields: `name: str`, `value: str`

3. **Operation type models** (per D-01: one Pydantic model per operation type):
   - `AddComponentOp(BaseModel)`:
     - `op_type: Literal["add_component"] = "add_component"`
     - `target_file: str` (per D-03: single file per operation)
     - `library_id: str = Field(description="Library reference, e.g. 'Device:R_Small_US'")`
     - `reference: str = Field(default="R?", description="Reference designator")`
     - `value: str = Field(default="", description="Component value")`
     - `position: PositionSpec`
   - `RemoveComponentOp(BaseModel)`:
     - `op_type: Literal["remove_component"] = "remove_component"`
     - `target_file: str`
     - `reference: str = Field(description="Reference designator to remove")`
   - `MoveComponentOp(BaseModel)`:
     - `op_type: Literal["move_component"] = "move_component"`
     - `target_file: str`
     - `reference: str`
     - `position: PositionSpec`
   - `ModifyPropertyOp(BaseModel)`:
     - `op_type: Literal["modify_property"] = "modify_property"`
     - `target_file: str`
     - `reference: str`
     - `property_name: str`
     - `new_value: str`

4. **Discriminated union** (per D-01):
   ```python
   class Operation(BaseModel):
       """Discriminated union of all operation types.
       
       Per D-02: each operation is atomic (one mutation).
       Per D-03: each operation targets one file via target_file.
       Per D-04: export full JSON Schema via model_json_schema().
       """
       root: Annotated[
           AddComponentOp | RemoveComponentOp | MoveComponentOp | ModifyPropertyOp,
           Field(discriminator="op_type"),
       ]
   ```
   Use `typing.Annotated` with `Field(discriminator="op_type")` for Pydantic v2 discriminated union support.

5. **Schema export helper function**:
   ```python
   def get_operation_schema() -> dict:
       """Export the full JSON Schema for LLM consumption (D-04)."""
       return Operation.model_json_schema()
   ```

Import order: stdlib (typing.Literal, typing.Annotated), third-party (pydantic), project-internal (none needed here).

CRITICAL: Every operation model MUST have `op_type: Literal["..."]` as a field with a unique value. Missing this on any model causes PydanticUserError at class definition time (Pitfall 1 from RESEARCH.md).
