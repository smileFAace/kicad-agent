7. Add `undo()` method to OperationExecutor:
```python
def undo(self, target_file: Optional[str] = None) -> dict[str, Any]:
    """Undo the most recent mutation for a file.

    Args:
        target_file: Relative path to file, or None for most recently modified.

    Returns:
        Dict with success, undone operation type, and target file.
    """
    if self._undo_stack is None:
        return {"success": False, "error": "Undo stack not enabled"}

    if target_file is not None:
        file_path = (self._base_dir / target_file).resolve()
        entry = self._undo_stack.pop_undo(file_path)
    else:
        entry = self._undo_stack.pop_latest_undo()

    if entry is None:
        return {"success": False, "error": "No operations to undo"}

    # Restore pre-mutation content
    entry.file_path.write_text(entry.pre_content, encoding="utf-8")

    # Invalidate cache for this file
    if self._cache:
        self._cache.invalidate(entry.file_path)

    return {
        "success": True,
        "undone_op": entry.op_type,
        "target_file": str(entry.file_path.relative_to(self._base_dir)),
    }
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| In-memory undo stack -> file system | undo/redo writes content from memory to disk (no external input) |
| Concurrent threads -> undo stack | Multiple threads may push/pop concurrently (mitigated by Lock) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-01 | Denial of Service | UndoStack | mitigate | Bounded deque(maxlen=max_size) prevents unbounded memory growth. Default 50, configurable via KICAD_UNDO_MAX_SIZE |
| T-33-02 | Tampering | undo()/redo() | mitigate | File path resolved and validated against project directory before write (existing path confinement in executor) |
| T-33-03 | Tampering | Stale snapshots | accept | If file modified externally after snapshot captured, undo restores stale content. Same risk as any editor undo. Low probability in MCP context. |
| T-33-04 | Information Disclosure | UndoStack memory | accept | Undo entries stored in process memory only, not persisted. No exposure surface beyond process memory. |
</threat_model>
