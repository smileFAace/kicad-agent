    **Part B: Create erc_auto_fix.py in src/kicad_agent/ops/**

    This is a new module. Structure:

    1. Define `VIOLATION_REPAIR_MAP` dict mapping violation type strings to repair function names:
       ```python
       VIOLATION_REPAIR_MAP: dict[str, str] = {
           "pin_not_connected": "place_no_connects",
           "power_pin_not_driven": "add_power_flag",
           "pin_to_pin": "fix_pin_type_mismatches",
           "missing_power_pin": "place_missing_units",
       }
       ```
       Note: break_wire_shorts and snap_to_grid are handled by checking violation message patterns, not exact type strings. Wire shorts are detected via "short" in the violation message. Off-grid is detected via "off_grid" in the type or message.

    2. Define priority order as a list of violation types. Shorts are highest priority (they cause the most damage), then pin type fixes, then power flags, then cosmetic (no-connects, off-grid):
       ```python
       REPAIR_PRIORITY = [
           "break_wire_shorts",     # shorts cause cascading errors
           "fix_pin_type_mismatches",  # type conflicts
           "place_missing_units",   # missing units
           "add_power_flag",        # power pin issues
           "place_no_connects",     # unconnected pins (cosmetic)
           "snap_to_grid",          # off-grid (cosmetic)
       ]
       ```

    3. Implement `erc_auto_fix(ir: SchematicIR, file_path: Path, max_iterations: int = 3) -> dict[str, Any]`:
       - Import parse_erc lazily from erc_parser
       - Loop for max_iterations:
         a. Call parse_erc(file_path) to get violations
         b. If no violations, break
         c. Group violations by type
         d. For each violation type in priority order:
            - Look up repair function in VIOLATION_REPAIR_MAP
            - If mapped, call the corresponding repair function with (ir, file_path)
            - If not mapped, add to unhandled list
         e. After repairs, check if total violation count decreased
         f. If count did not decrease (or increased), break early
       - Serialize the schematic after all repairs
       - Return dict with:
         ```python
         {
             "fixes_applied": [{"type": "pin_not_connected", "count": 3, "repair": "place_no_connects"}, ...],
             "iterations": iteration_count,
             "remaining_violations": len(violations_after),
             "unhandled_violations": [{"type": "unknown_type", "count": 1}, ...],
         }
         ```

    4. The repair function calls should import and call the actual repair implementations directly (not re-dispatch through executor). The functions in repair.py and the repair modules are regular Python functions that accept (ir, file_path, ...).

    CRITICAL: Per RESEARCH Pitfall 5 -- enforce max_iterations, track violation counts, stop early if no decrease. Never loop infinitely.

    CRITICAL: The repair functions modify `ir` in-place (they call `ir._record_mutation`). After all repairs, the executor will serialize `ir` via the normal Transaction flow (schematic handler path).
