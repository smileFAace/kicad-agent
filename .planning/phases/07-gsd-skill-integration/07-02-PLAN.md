```python
@dataclass(frozen=True)
class OperationResult:
    success: bool
    operation_type: str  # e.g. "add_component"
    target_file: str
    message: str
    details: dict[str, Any]  # operation-specific details

@dataclass(frozen=True)
class OperationError:
    success: bool  # always False
    operation_type: str  # "unknown" if parse failed
    error: str
    suggestion: str  # actionable next step
```

Add a `to_text()` method to both that renders a human-readable string suitable for Claude to display to the user. OperationResult.to_text() shows success + message + details. OperationError.to_text() shows error + suggestion.

**src/kicad_agent/handler.py** -- Handler routing:

1. `validate_operation(json_str: str) -> tuple[Operation | None, OperationError | None]`:
   - Parse JSON string into a dict via json.loads()
   - Validate against Operation.model_validate({"root": parsed_dict})
   - Return (Operation, None) on success, (None, OperationError) on failure
   - Catch json.JSONDecodeError, pydantic.ValidationError, and generic exceptions
   - Each error type produces a specific suggestion (e.g., "Check JSON syntax" vs "Check operation fields against schema")

2. `handle_operation(json_str: str, project_dir: Path | None = None) -> OperationResult | OperationError`:
   - Call validate_operation() first
   - If validation fails, return the OperationError immediately
   - Dispatch based on op_type:
     - For now, return an OperationResult indicating the operation was validated and queued
     - Include a note that actual mutation dispatch will be wired in Phase 4+ when operation executors exist
   - The handler must NOT attempt mutations yet -- it validates and routes, returning a structured response

3. `format_result(result: OperationResult | OperationError) -> str`:
   - Call result.to_text() and return the formatted string

Do NOT import mutation-specific modules. The handler validates and prepares operations; actual IR mutations are Phase 4+. The handler must work with only the schema module and standard library.</action>
